the rules of the game

01 / PRE-BATTLE & ARENA ENGAGEMENT
pre_battle_rules.md
**PRE-BATTLE SETUP** 1. chaos-team-builder deploys 5-agent chaos team (chaos-strategist, infra/network/app-chaos, observer-chaos) 2. sre-team-lead activates all 9 SME agents 3. SRE writes postmortem → runbooks → pushed to GitHub 4. Both teams briefed: no inter-team communication 5. Both teams confirm ready **THE BATTLE** *(coordinator sends milestone updates only)* 6. Phase 1: reconnaissance (~5 min) → *Coordinator: 'Phase 1 begun'* 7. Phase 2: initial attack executes directly → *Coordinator: 'Phase 2 begun'* 8. SRE detects, diagnoses, remediates directly → *Coordinator: 'SRE remediated X of Y'* 9. Phase 3+: compound attacks if applicable → *Coordinator: 'Phase 3 begun'* 10. Exercise ends when playbook exhausted or SRE fully remediates
Playbook Lifecycle
[SETUP] Teams Activated & Baseline Config Checked
[PHASE 1] Attackers scan cluster & VPC networking (Recon)
[PHASE 2] Automated attack vectors execute / SRE triage begins
[REMEDIATION] SRE writes code, sweeps selectors, scales pods
02 / POST-BATTLE CONTINUOUS LEARNING LOOP
post_battle_loop.md
**POST-BATTLE** 11. Full cleanup verified by SRE team 12. observer-chaos scores, both teams debrief 13. **SRE team extracts learnings:** - New skills/detection gaps → updated SME skill docs - Improved runbooks → pushed to GitHub - Agent template improvements → team-builder updates SME templates - Goal: each battle makes the SRE team measurably better 14. 3 artifacts published to GCS and sent to Cedric
